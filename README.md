Full Stack Engineer focado em sistemas de operação real: ERP, financeiro, backoffice, dashboards e produtos com IA.

### salomao

Sistema financeiro full stack rodando em VPS, com módulos de cobrança, conciliação, compras, fluxo de caixa, DRE/DRO, projeções e integração com o Banco Inter (extrato, boleto, baixa, cancelamento).

- Front em React 18, TypeScript e Vite.
- Back em FastAPI, SQLAlchemy e Pydantic, com PostgreSQL e Alembic.
- Redis para cache de analytics e snapshots históricos.
- MFA, cookies HttpOnly, rate limit, criptografia de campos, audit logs.
- Deploy em VPS com Nginx, systemd, UFW, fail2ban, HTTPS.
- CI/CD em GitHub Actions com self-hosted runner, branch `dev` para homologação e `main` para produção.

### growth-agent

Base SaaS de growth content com IA, automação de publicação e aprovação humana antes de qualquer post sair.

- Next.js 16, React 19 e App Router.
- Prisma, PostgreSQL, Zod e Clerk.
- AI SDK, OpenAI e Trigger.dev para jobs assíncronos.
- Separação clara entre operação de clientes e laboratório interno de experimentos.

### meu-negocio-facil-next

ERP acadêmico onde atuei como Lead Developer e Scrum Master, com 225+ commits próprios. PDV, estoque, vendas, autenticação, dashboards e estado global.

## Como eu trabalho

Especificação antes de código. O fluxo costuma ser:

1. Entender a regra de negócio.
2. Escrever uma spec com contratos, entidades, cálculos e critérios de aceite.
3. Quebrar em tarefas pequenas, executar (sozinho ou com agentes) e revisar.
4. Validar em ambiente próximo de produção.

Uso IA como acelerador, mas arquitetura, segurança e decisão de produto continuam minha responsabilidade.
